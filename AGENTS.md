## Quick Start Commands
Run from repository root unless noted.

```bash
composer install
yii init
yii migrate --migrationPath=@yii/rbac/migrations/
yii migrate
yii auth/init
```

Then install frontend assets from the path used by this project:

```bash
cd backend/web/vendor
npm install
```

## Testing Commands
```bash
yii_test
codeception run --config backend
codeception run --config frontend
codeception run --config common
```

## Architecture Boundaries
- `common/`: shared code and cross-app configuration.
  - Put shared ActiveRecord/domain models in `common/models/`.
  - Reusable services live in `common/components/` and are registered in `common/config/main.php`.
- `backend/`: admin web app (`backend/web/index.php` entrypoint).
- `frontend/`: public/user web app (`frontend/web/index.php` entrypoint).
- `console/`: CLI commands and migrations.
- `environments/`: templates used by `yii init` for local/prod config files.
- `vendor/`: third-party dependencies, do not edit manually.

## Conventions For Changes
- Follow Yii MVC defaults:
  - Controllers in `*/controllers/*Controller.php`
  - Views in matching `*/views/<controller>/<action>.php`
  - Search models use `<Model>Search` naming.
- Prefer placing business logic in models/components, not in views.
- Keep environment secrets and machine-specific values in `*-local.php` files generated by `yii init`.
- Do not commit generated runtime or cache artifacts (`*/runtime`, `*/web/assets`).

## Operational Pitfalls
- `yii init` is required after fresh checkout to materialize local config and cookie validation keys.
- Writable directories are defined in [environments/index.php](environments/index.php). If app boot fails, check permissions for:
  - `backend/runtime`, `backend/web/assets`
  - `frontend/runtime`, `frontend/web/assets`
  - `console/runtime`
- Docker defaults use MySQL 5.7 credentials from [docker-compose.yml](docker-compose.yml); treat these as development defaults only.
- Repository currently has no `docs/` directory even though [README.md](README.md) references `docs/guide/README.md`.

## Agent Guardrails
- Make minimal, targeted edits; preserve existing style and folder structure.
- Do not refactor across `backend`, `frontend`, `common`, `console` unless requested.
- When adding features that touch DB schema, add a migration in `console/migrations/` and mention required migrate command.
- Prefer validated commands from this file and [README.md](README.md) over assumptions.
