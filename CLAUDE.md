## Project Overview

**Profility** is a loyalty/rewards point collection system ("Raccolta Punti") built on **Yii2 Advanced Template**. It supports receipt scanning via OCR (Asprise), QR code generation, campaign management, and role-based access.

## Quick Start

```bash
composer install
yii init                                        # generates local config and cookie keys (required after fresh checkout)
yii migrate --migrationPath=@yii/rbac/migrations/
yii migrate
yii auth/init

cd backend/web/vendor && npm install            # frontend assets
```

## Testing

```bash
yii_test                                        # all suites
vendor/bin/codeception run --config backend     # backend only
vendor/bin/codeception run --config frontend    # frontend only
vendor/bin/codeception run --config common      # common only
```

Test config for backend requires `backend/config/codeception-local.php` (generated by `yii init`).

## Architecture

Three-tier Yii2 Advanced application:

| Directory | Purpose |
|-----------|---------|
| `common/` | Shared ActiveRecord models, components, config — used by all tiers |
| `backend/` | Admin web app — entry point `backend/web/index.php` |
| `frontend/` | Public/user web app — entry point `frontend/web/index.php` |
| `console/` | CLI commands and DB migrations |
| `environments/` | Templates used by `yii init` for local/prod config files |

**Config merge chain** (applied at boot): `common/config/main.php` → `common/config/main-local.php` → `{app}/config/main.php` → `{app}/config/main-local.php`. Secrets and machine-specific values belong in `*-local.php` files only.

### Key Domain Models (`common/models/`)

- `User` — identity/auth
- `Profilo` — user profile
- `Scontrino` — receipt/transaction
- `Puntovendita` — point-of-sale location
- `Premio` — reward catalog
- `Campagna` — promotion/campaign
- `Coupon` — coupon management
- `Configuration` — system settings

### Key Components (`common/components/`)

- `GestorePunti` — points crediting logic
- `QRCode` — QR code generation (`chillerlan/php-qrcode`)
- `Asprise` — OCR integration for receipt scanning
- `ScontrinoHelper` — receipt processing
- `MailerHelper` — email utilities (registered in `common/config/main.php`)

### RBAC Roles

`admin`, `business`, `simpleuser`, `direzione` — initialized via `yii auth/init`.

## Conventions

- Follow Yii MVC defaults: controllers as `*Controller.php`, views under `*/views/<controller>/<action>.php`, search models as `<Model>Search`.
- Business logic goes in models/components, not views.
- When touching DB schema, add a migration in `console/migrations/` and note the required `yii migrate` command.
- Do not refactor across `backend`/`frontend`/`common`/`console` unless explicitly requested.
- Make minimal, targeted edits; preserve existing style and folder structure.

## Operational Notes

- `backend/runtime`, `backend/web/assets`, `frontend/runtime`, `frontend/web/assets`, `console/runtime` must be writable — see `environments/index.php`.
- Docker dev environment: `docker-compose up` (frontend `:20080`, backend `:21080`, MySQL `:3306`). DB credentials in `docker-compose.yml` are dev-only defaults.
- Do not commit `*/runtime/` or `*/web/assets/` artifacts.
